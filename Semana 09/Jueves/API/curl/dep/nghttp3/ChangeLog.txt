    cmake: Fix linker warning LNK4291 when linking static lib to the final binary
    
    Apparently some functions (e.g. nghttp3_balloc_clear) create xdata sections in the lib. These are used on 64bit platforms to judge which control flow transitions should be allowed. Normally they are needed only in uncommon cases like longjmp, SEH exceptions or SetThreadContext Windows API calls. These functions don't do anything to warrant it. This flag should just let the compiler do it's thing more efficiently.
